# telerithm-playground

A demo app that shows telerithm in action — the way real users integrate it.

**Live:** https://play.telerithm.cloud

## Concept

Most monitoring tools show you a screenshot. We show you the real thing.

`play.telerithm.cloud` is a fake but realistic web app (an e-commerce store or SaaS dashboard) that has the **telerithm SDK installed** — exactly like a real user would integrate it. When you interact with the app, real logs flow into telerithm in real time.

```
play.telerithm.cloud              demo.telerithm.cloud
┌────────────────────────┐        ┌─────────────────────────┐
│  Demo App              │        │  Telerithm Dashboard    │
│  (SDK integrated)      │──────→ │                         │
│                        │  logs  │  Live-Tail  AI Query    │
│  [Login] → fail        │        │  Alerts     Issues      │
│  [Checkout] → slow     │        │                         │
│  [API call] → error    │        │  "Show payment errors   │
│                        │        │   from last hour"  →    │
│  "See this in          │        │  SQL + results          │
│   Telerithm →"         │        └─────────────────────────┘
└────────────────────────┘
```

## User Journey

1. User opens `play.telerithm.cloud`
2. Sees a realistic demo app with interactive scenarios
3. Clicks: "Simulate failed checkout", "Trigger auth error", "Generate traffic spike"
4. Sees a counter: "3 logs sent to Telerithm"
5. Clicks "Open in Telerithm →" → lands on `demo.telerithm.cloud` (auto-joined demo team)
6. Sees their logs live — in a real dashboard, with real AI query, real alerts

## Scenarios

The demo app simulates realistic production situations:

| Scenario | What Happens | Logs Generated |
|----------|-------------|----------------|
| Failed Login | User tries to log in, fails 3x, gets locked out | 5 error logs (auth-service) |
| Slow Checkout | Payment API times out, retry, success | 8 warn/error logs (payment-service) |
| API Error Spike | 50 requests, 10% return 500 | 50 mixed logs (api-gateway) |
| Deploy Event | App "deploys" new version, old pod drains | 15 info logs (infra) |
| Normal Traffic | Background activity, healthy | 100 info logs across 3 services |

## Architecture

### play.telerithm.cloud
- Next.js app (consistent with telerithm)
- Has `@telerithm/sdk-js` installed and initialized
- SDK points to demo team source (pre-configured API key)
- Shows scenario buttons + live counter of sent logs
- Link to open demo.telerithm.cloud

### demo.telerithm.cloud
- The actual telerithm app
- Pre-seeded demo team with realistic historical logs
- Auto-join: new visitors get read-only demo access on first visit
- Shows the logs generated by play in real time

### Integration example visible in UI
The playground shows a code snippet of how the SDK is used:
```typescript
import { TelerithmClient } from "@telerithm/sdk-js";
const t = new TelerithmClient({ dsn: "https://key@demo.telerithm.cloud/source" });
t.error("Payment failed", { userId: "u_123", amount: 99.00 });
```

This teaches users how to integrate while they play.
